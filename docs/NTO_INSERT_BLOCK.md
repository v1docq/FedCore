# Блок для вставки в заключительный НТО

## Сведения о публикации открытой библиотеки FedCore

По результатам выполнения работ подготовлена и опубликована открытая библиотека **FedCore**, предназначенная для автоматизации создания, сжатия, адаптации и портирования моделей машинного обучения на основе глубоких нейронных сетей. Библиотека размещена в публичном репозитории в сети Интернет по адресу:

```text
https://github.com/v1docq/FedCore
```

Доступ к использованию библиотеки предоставляется на условиях открытой лицензии **BSD 3-Clause License**. Текст лицензии размещён в корне публичного репозитория:

```text
https://github.com/v1docq/FedCore/blob/main/LICENSE
```

Русскоязычная инструкция по установке, запуску и базовому использованию библиотеки размещена в разделе документации:

```text
https://github.com/v1docq/FedCore/blob/main/docs/tutorial_ru.md
```

Документация и материалы для пользователя размещены по адресу:

```text
https://github.com/v1docq/FedCore/tree/main/docs
```

В репозитории также представлены примеры новых вариантов использования кода, включая:

1. сжатие модели ResNet-18 методом pruning;
2. малоранговое разложение модели ResNet-18;
3. квантизацию модели ResNet-18;
4. экспорт модели в формат ONNX;
5. запуск вспомогательного модуля экспорта моделей через Docker.

Примеры размещены по следующим адресам:

```text
https://github.com/v1docq/FedCore/tree/main/examples/pruning_resnet18
https://github.com/v1docq/FedCore/tree/main/examples/low_rank_resnet18
https://github.com/v1docq/FedCore/tree/main/examples/quantization_resnet18
https://github.com/v1docq/FedCore/tree/main/examples/export_onnx
https://github.com/v1docq/FedCore/tree/main/deployment/model_exporter
```

Для каждого примера подготовлены пошаговые инструкции по запуску, демонстрационные скрипты и/или Jupyter Notebook-файлы с пояснением ожидаемых результатов. Примеры позволяют проверить основные пользовательские сценарии библиотеки: применение методов сжатия, оценку изменения размера модели и времени инференса, экспорт модели во внешний формат и запуск вспомогательного сервиса экспорта через контейнеризацию.

Минимальные технические требования для базового запуска библиотеки включают: операционную систему Linux Ubuntu 20.04/22.04 или совместимую ОС, Python версии 3.8 и выше, 8 ГБ оперативной памяти, x86-64 CPU, git, pip/venv или conda. Для GPU-сценариев рекомендуется наличие NVIDIA GPU, драйвера NVIDIA, совместимой версии CUDA и PyTorch с поддержкой CUDA. Для контейнерного запуска демонстрационного модуля экспорта используются Docker и Docker Compose.
