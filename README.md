## О библиотеке

**FedCore** — открытая библиотека для автоматизации создания, сжатия, адаптации и портирования моделей машинного обучения на основе глубоких нейронных сетей.

Библиотека предназначена для:

- снижения вычислительной сложности моделей;
- уменьшения объёма памяти, занимаемой весами модели;
- подготовки моделей к запуску на различных вычислительных архитектурах;
- автоматизации применения методов прунинга, квантизации и малорангового разложения;
- оценки качества, размера модели и времени инференса;
- экспорта моделей в форматы, пригодные для запуска на целевых аппаратных платформах.

## Назначение библиотеки

FedCore предоставляет инструменты для:

1. сжатия моделей машинного обучения;
2. применения методов прунинга, квантизации и малорангового разложения;
3. автоматизированного подбора стратегии адаптации модели;
4. оценки качества, размера модели и времени инференса;
5. экспорта моделей в форматы, пригодные для запуска на различных аппаратных платформах;
6. подготовки моделей к использованию на edge-устройствах и специализированных вычислителях.

---

## Основные возможности

Библиотека включает следующие функциональные модули:

- модуль прунинга нейронных сетей;
- модуль квантизации моделей;
- модуль малорангового разложения весов;
- модуль эволюционной оптимизации стратегий сжатия;
- API для запуска сценариев обучения, адаптации, оценки и экспорта моделей;
- инструменты оценки качества и вычислительных характеристик моделей;
- инструменты экспорта моделей в целевые форматы;
- Docker-конфигурации для развертывания библиотеки и вспомогательных сервисов.

## Направления прикладного использования

Библиотека FedCore может применяться в следующих прикладных направлениях:

1. промышленные IoT-системы с ограничениями по памяти и вычислительной мощности;
2. автономные системы навигации и детекции объектов на беспилотных платформах;
3. системы компьютерного зрения на edge-устройствах;
4. системы распознавания объектов в условиях ограниченного энергопотребления;
5. системы обработки потоковых данных с низкой задержкой;
6. системы, требующие компактных моделей для передачи по ограниченным каналам связи;
7. системы прогнозирования энергопотребления;
8. промышленные системы мониторинга и диагностики;
9. мобильные и встраиваемые приложения машинного обучения;
10. задачи портирования моделей на специализированные аппаратные архитектуры.

## Минимальные технические требования

Для базового запуска библиотеки необходима следующая программно-аппаратная среда.

| Компонент | Минимальное требование | Рекомендуемое значение |
|---|---|---|
| Операционная система | Linux Ubuntu 20.04 / 22.04 или совместимая ОС | Ubuntu 22.04 |
| Python | 3.8+ | 3.10 |
| Оперативная память | 8 ГБ | 16 ГБ и более |
| Процессор | x86-64 CPU | многоядерный x86-64 CPU |
| GPU | не требуется для базовых сценариев | NVIDIA GPU с CUDA |
| Система контроля версий | git | актуальная версия git |
| Пакетный менеджер | pip | pip + venv / conda |
| Контейнеризация | не требуется | Docker, Docker Compose |

Для запуска GPU-сценариев дополнительно рекомендуется:

- NVIDIA GPU с поддержкой CUDA;
- установленный драйвер NVIDIA;
- совместимая версия CUDA;
- PyTorch с поддержкой CUDA.

Для сценариев экспорта и запуска моделей на целевых аппаратных архитектурах могут потребоваться дополнительные SDK и фреймворки производителей, например TensorRT, RKNN Toolkit2, ONNX Runtime, LiteRT / TFLite, OpenVINO, TVM или иные инструменты, соответствующие целевой платформе.

## Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/v1docq/FedCore.git
cd FedCore
```

### 2. Создание виртуального окружения

Для Linux / macOS:

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Установка библиотеки в режиме разработки

```bash
pip install -e .
```

Для запуска демонстрационных ноутбуков дополнительно могут потребоваться:

```bash
pip install notebook pandas matplotlib onnx fastapi uvicorn python-multipart
```

## Быстрый старт

Ниже приведён базовый сценарий использования FedCore для сжатия модели и получения отчёта по метрикам.

```python
from fedcore.api.main import FedCore

# Пример является демонстрационным.
# Конкретная конфигурация зависит от выбранной модели,
# датасета и метода сжатия.

fedcore = FedCore(api_config)

fedcore.fit(train_data)

report = fedcore.get_report(test_data)

print(report)
```

## Примеры новых вариантов использования кода

В репозитории представлены примеры использования библиотеки для разных сценариев. Каждый пример сопровождается пошаговой инструкцией по запуску.

### Пример 1. Сжатие модели ResNet-18 методом pruning

**Назначение:** показать применение прунинга к модели ResNet-18 и оценить изменение качества, размера модели и времени инференса.

**Путь к примеру:**

```text
examples/pruning_resnet18/
```

**Инструкция по запуску скрипта:**

```bash
cd examples/pruning_resnet18
python run_pruning_resnet18.py
```

**Инструкция по запуску ноутбука:**

```bash
jupyter notebook examples/pruning_resnet18/resnet18_pruning_demo.ipynb
```

**Ожидаемый результат:**

- загрузка или подготовка модели ResNet-18;
- применение метода pruning;
- оценка качества модели до и после сжатия;
- вывод размера модели до и после сжатия;
- вывод времени инференса;
- сохранение отчёта в `results/pruning_resnet18/metrics.csv`.

### Пример 2. Малоранговое разложение модели

**Назначение:** показать применение low-rank decomposition к слоям нейронной сети и последующую оценку вычислительных характеристик модели.

**Инструкция по запуску ноутбука:**

**Ожидаемый результат:**

- применение малорангового разложения;

```text
http://localhost:5000
```

Проверка состояния сервиса:

```bash
curl http://localhost:5000/health
```

## REST API модуля экспорта моделей

При запуске модуля экспорта моделей доступны следующие основные эндпоинты.

| Эндпоинт | Метод | Назначение |
|---|---|---|
| `/` | GET | Web-интерфейс модуля |
| `/health` | GET | Проверка состояния сервиса |
| `/files` | GET | Получение списка загруженных и экспортированных файлов |
| `/upload` | POST | Загрузка файлов |
| `/export` | POST | Демонстрационный экспорт модели в выбранный формат |
| `/analyze_model` | POST | Анализ загруженного модельного файла |
