# Руководство пользователя FedCore

Данный документ содержит пошаговую инструкцию по установке, запуску и базовому использованию открытой библиотеки **FedCore**.

FedCore — библиотека для автоматизации сжатия, адаптации и портирования моделей машинного обучения на основе глубоких нейронных сетей. Библиотека позволяет применять методы прунинга, квантизации и малорангового разложения, оценивать качество и вычислительные характеристики моделей, а также готовить модели к запуску на различных вычислительных архитектурах.

## 2. Минимальные технические требования

Для базового запуска библиотеки требуется:

| Компонент | Минимальное требование | Рекомендуемое значение |
|---|---|---|
| Операционная система | Linux Ubuntu 20.04 / 22.04 или совместимая ОС | Ubuntu 22.04 |
| Python | 3.10+ | 3.10–3.12 |
| Оперативная память | 8 ГБ | 16 ГБ и более |
| Процессор | x86-64 CPU | многоядерный x86-64 CPU |
| GPU | не требуется для базовых сценариев | NVIDIA GPU с поддержкой CUDA |
| Git | требуется | актуальная версия |
| pip / venv | требуется | актуальная версия |
| Docker | не требуется для базового запуска | Docker и Docker Compose |

Для GPU-сценариев дополнительно рекомендуется установить:

- драйвер NVIDIA;
- совместимую версию CUDA;
- PyTorch с поддержкой CUDA.

Для экспорта моделей под отдельные аппаратные платформы могут потребоваться дополнительные инструменты и SDK: TensorRT, RKNN Toolkit2, ONNX Runtime, LiteRT / TFLite, OpenVINO, TVM или иные средства, соответствующие целевой архитектуре.

## 3. Установка библиотеки

### 3.1. Клонирование репозитория

```bash
git clone https://github.com/v1docq/FedCore.git
cd FedCore
```

### 3.2. Создание виртуального окружения

Для Linux / macOS:

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3.3. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 3.4. Установка библиотеки в режиме разработки

```bash
pip install -e .
```

После этого библиотека доступна для импорта из Python-кода.

## 4. Проверка установки

Для проверки установки можно выполнить команду:

```bash
python -c "import fedcore; print('FedCore installed successfully')"
```

Если команда завершается без ошибки, библиотека установлена корректно.

## 5. Базовая логика работы с FedCore

Типовой сценарий работы с библиотекой включает следующие этапы:

1. подготовка исходной модели и данных;
2. выбор метода сжатия или адаптации модели;
3. настройка конфигурации эксперимента;
4. запуск процедуры сжатия / адаптации;
5. оценка качества и вычислительных характеристик;
6. сохранение или экспорт полученной модели.

На уровне API работа обычно строится вокруг объекта `FedCore`, который получает конфигурацию эксперимента, выполняет обучение / адаптацию модели и формирует отчёт с метриками.

Упрощённая схема:

```python
from fedcore.api.main import FedCore

fedcore = FedCore(api_config)

fedcore.fit(train_data)

report = fedcore.get_report(test_data)

print(report)
```

Конкретный состав `api_config`, `train_data` и `test_data` зависит от выбранной задачи, модели и метода сжатия.

## 6. Пример 1. Запуск прунинга модели

Прунинг используется для удаления наименее значимых параметров модели и уменьшения вычислительной сложности.

В репозитории уже предусмотрены примеры для задач компьютерного зрения и временных рядов. Для запуска примера с прунингом можно использовать один из скриптов в каталоге:

```text
examples/api_example/pruning/
```

Например, для задачи компьютерного зрения:

```bash
cd examples/api_example/pruning/cv_task
python prune_conv_model.py
```

Ожидаемый результат:

1. загрузка или создание исходной модели;
2. применение выбранной стратегии прунинга;
3. дообучение модели при необходимости;
4. вывод метрик качества;
5. оценка изменения размера и вычислительных характеристик модели.

Если для конкретного примера требуется предварительно обученная модель, необходимо проверить наличие файлов в каталоге `pretrain_models` или указать собственный путь к модели в параметрах скрипта.

## 7. Пример 2. Запуск малорангового разложения

Малоранговое разложение используется для аппроксимации матриц весов модели через представление меньшего ранга. Это позволяет уменьшить количество параметров и объём памяти, необходимый для хранения модели.

Примеры расположены в каталоге:

```text
examples/api_example/low_rank/
```

Запуск примера для ResNet:

```bash
cd examples/api_example/low_rank
python cifar_resnet.py
```

Ожидаемый результат:

1. загрузка модели;
2. применение малорангового разложения;
3. формирование сжатого представления слоёв;
4. оценка качества модели после преобразования;
5. вывод сравнительных метрик.

## 8. Пример 3. Запуск квантизации модели

Квантизация используется для перехода от стандартного представления параметров модели к более компактным числовым форматам, например INT8 или FP16.

Пример расположен в каталоге:

```text
examples/api_example/quantization/cv_task/
```

Запуск:

```bash
cd examples/api_example/quantization/cv_task
python quant_conv_model.py
```

Ожидаемый результат:

1. загрузка модели;
2. применение квантизации;
3. сохранение или использование квантизированной модели;
4. сравнение размера модели до и после квантизации;
5. оценка качества и времени инференса.

## 9. Пример 4. Использование ModelRegistry

ModelRegistry используется для управления жизненным циклом моделей, хранения промежуточных состояний и работы с версиями моделей до и после сжатия.

Примеры расположены в каталоге:

```text
examples/api_example/model_registry_example/
```

Запуск примера для обучения модели ResNet:

```bash
cd examples/api_example/model_registry_example/training
python resnet_example.py
```

Запуск примера для малорангового разложения:

```bash
cd examples/api_example/model_registry_example/low_rank
python resnet_example.py
```

Ожидаемый результат:

1. создание или загрузка модели;
2. регистрация модели в хранилище;
3. выполнение выбранного сценария обработки;
4. сохранение результатов и логов.

## 10. Пример 5. Экспорт модели

Экспорт модели используется для подготовки модели к запуску на целевой вычислительной архитектуре или во внешнем фреймворке.

Типовой сценарий экспорта включает:

1. загрузку обученной или сжатой модели;
2. выбор целевого формата;
3. запуск процедуры экспорта;
4. проверку корректности экспортированного файла;
5. сохранение результата в директорию экспорта.

Поддерживаемые или предполагаемые форматы экспорта могут включать:

- ONNX;
- TorchScript;
- TensorRT;
- TFLite / LiteRT;
- OpenVINO;
- TVM;
- другие форматы в зависимости от целевой архитектуры.

Общий пример логики экспорта:

```python
converted_model = fedcore.export(
    framework="ONNX",
    framework_config=framework_config,
    supplementary_data=supplementary_data
)
```

Конкретные параметры `framework_config` и `supplementary_data` зависят от модели и целевого формата.

## 11. Запуск через Docker

Если в репозитории предусмотрен Dockerfile, библиотеку можно запустить в контейнере.

Сборка образа:

```bash
docker build -t fedcore .
```

Запуск контейнера:

```bash
docker run --rm -p 3000:3000 fedcore
```

Если используется `docker-compose.yml`:

```bash
docker compose up --build
```

После запуска веб-интерфейс или API, при наличии соответствующей конфигурации, будут доступны по порту, указанному в Dockerfile или docker-compose.yml.

## 12. Возможные проблемы и способы их устранения

### Ошибка: зависимости не установлены

Решение:

```bash
pip install --upgrade pip
pip install -r requirements.txt
pip install -e .
```

### Ошибка: модуль `fedcore` не найден

Решение: убедиться, что команда запуска выполняется из активированного виртуального окружения, а библиотека установлена в режиме разработки:

```bash
source .venv/bin/activate
pip install -e .
```

Для Windows:

```bash
.venv\Scripts\activate
pip install -e .
```

### Ошибка при запуске GPU-сценария

Решение:

1. проверить наличие NVIDIA GPU;
2. проверить установленный драйвер NVIDIA;
3. проверить совместимость CUDA и PyTorch;
4. при необходимости запустить CPU-сценарий.

Проверка доступности CUDA в PyTorch:

```bash
python -c "import torch; print(torch.cuda.is_available())"
```

### Ошибка при экспорте модели

Возможные причины:

1. целевой формат не поддерживает отдельные операции модели;
2. отсутствует SDK целевой платформы;
3. некорректно задан пример входного тензора;
4. используется неподдерживаемая версия opset для ONNX.

Рекомендуется:

1. проверить лог экспорта;
2. попробовать другой формат экспорта;
3. обновить зависимости;
4. проверить ограничения целевой аппаратной платформы.
